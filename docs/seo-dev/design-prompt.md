# Промт для нейросети-дизайнера: макеты SEO-раздела «Проверка сайтов»

> Готовый к копипасту самодостаточный промт. Дизайнер не имеет доступа к репозиторию — все значения дизайн-системы приведены ниже полностью. Скопируйте всё содержимое между маркерами «НАЧАЛО ПРОМТА» и «КОНЕЦ ПРОМТА».

## НАЧАЛО ПРОМТА

### 1. Контекст и железные правила

Ты — продуктовый дизайнер. Проектируешь макеты для сайта «Проверка сайтов» (домен проверкасайтов.рф, язык — русский, рынок — РФ). На сайте живут **два сервиса**:

- **Сервис А (существующий)** — юридическая проверка сайта: автоматический скан на соответствие законам РФ (152-ФЗ, cookie-баннер, оферта и т.д.), 21 проверка. Бесплатный скан даёт балл из 100 + светофор + 2 критичных нарушения; полный отчёт стоит 700 ₽. Его главная страница уже существует и работает — её мы почти не трогаем.
- **Сервис Б (новый, его ты и проектируешь)** — SEO/технический аудит сайта: экспресс-аудит страницы (25+ проверок: мета-теги, скорость и Core Web Vitals, SSL, security-заголовки, robots.txt, sitemap и т.д.), набор отдельных бесплатных инструментов, платный полный аудит и платный мониторинг. Живёт в подкаталоге `/seo/` того же сайта.

Настроение дизайна — **«frosted glass cathedral at midnight»**: почти чёрный полуночный холст, фростед-стеклянные поверхности, тонкая blueprint-сетка на фоне, светящийся текст, будто подсвеченный из-за стеклянного слоя, и единственный хроматический акцент — яркий фиолетовый. Просторный, ритмичный, «соборный» воздух, а не плотный SaaS.

**Железное правило:** на сайте уже есть работающая дизайн-система, и весь дизайн-код новых страниц обязан **в точности** повторять её. НЕ вводи новые цвета, новые радиусы, новые шрифты, новые типы теней. Все значения даны ниже — используй только их. Любой новый оттенок, «улучшенный» градиент или дополнительный акцентный цвет ломает систему и будет отклонён.

#### 2.3. Радиусы — четыре семейства, не смешивать

| Элемент | Радиус |
|---|---|
| Кнопки (все, включая CTA) | 999px (pill) |
| Карточки и модалки | 16px |
| Бейджи и инпуты | 6px |
| Круглые иконочные контейнеры | 9999px (идеальный круг) |

Каждая кнопка — pill. Каждая карточка — 16px. Каждый бейдж и инпут — 6px. Никаких «слегка скруглённых» кнопок или квадратных карточек.

#### 2.4. Тени и elevation — только inset-фрост + ореол, БЕЗ обычных drop-shadow

Система читает высоту через внутреннюю фрост-подсветку верхней кромки + мягкое внутреннее свечение + тёмный холодный внешний ореол. Стандартные стеки (копировать буквально):

- **Hairline (универсальный «бордер»):** `inset 0 0 0 1px rgba(186,215,247,0.12)` — вместо любых сплошных бордеров.
- **Feature-карточка:** `inset 0 1px 1px rgba(199,211,234,0.12), inset 0 24px 48px rgba(199,211,234,0.05), 0 24px 32px rgba(6,6,14,0.7)`.
- **Модальная/формовая карточка:** `inset 0 1px 1px rgba(216,236,248,0.2), inset 0 24px 48px rgba(168,216,245,0.06), 0 16px 32px rgba(0,0,0,0.3)`.
- **Glow (ореол hero-уровня):** `0 0 6px rgba(186,207,247,0.32)` и `0 0 12px rgba(238,186,247,0.24)` — по эталону применяется ТОЛЬКО за hero-wordmark / крупнейшим display-заголовком; за другими крупными элементами — только тёмный внешний ореол из стеков карточек выше.
- **Glow фиолетового CTA:** `0 0 24px rgba(102,58,243,0.35)` — ⚠ санкционированное расширение системы для этого проекта: в исходной системе у Violet CTA тени нет, в эталонном списке теней этой строки не ищи. Применять только на главном фиолетовом CTA; других новых теней не вводить.

#### 2.5. Фон страницы

Полноэкранный слой атмосферы позади всех секций:

1. **Blueprint-сетка:** линии 1px цветом `rgba(186,215,247,0.06)`, шаг ячейки ~80–100px, полное покрытие, с маской-затуханием к краям (сетка гаснет книзу и по бокам).
2. **Spotlight:** конический градиентный ореол у верхнего центра страницы — `conic-gradient(at 50% -5%, transparent 45%, rgba(124,145,182,0.3) 49%, rgba(124,145,182,0.5) 50%, rgba(124,145,182,0.3) 51%, transparent 55%)` — эффект прожектора, освещающего hero сверху.

- Максимальная ширина контента: **1200px**, по центру, full-bleed тёмный холст вокруг.
- Отступ между секциями: **120px**.
- Внутренний паддинг карточек: **24px**.
- Базовый gap элементов: **16px**. Базовая единица сетки — 4px.
- Заголовочный блок секции: eyebrow → заголовок по центру heading-lg 44px (display 48px — только H1 хаба; heading 28px — только шапка отчёта и вложенные подсекции, см. §4) → одна строка приглушённого подзаголовка (16–18px), максимум ~640px ширины.

#### 2.8. Do / Don't

DO:
- 999px для всех интерактивных элементов; 16px только карточки/модалки; 6px только бейджи/инпуты; 9999px только круглые иконки.
- Elevation — только inset-фрост + внутренний glow + тёмный холодный внешний ореол.
- `#663af3` — единственная цветная КНОПКА (только главный CTA); как заливка state-индикаторов допускается ровно в двух местах — прогресс-бар скана и checked-чекбокс. Никогда как декоративный акцент.
- Display-заголовки — Space Grotesk вес 500, 44–48px; на самых крупных — Skywash-градиент.
- Каждую секцию открывать eyebrow с затухающими боковыми линиями.
- Универсальный бордер — hairline `rgba(186,215,247,0.12)` inset, никогда сплошной stroke.
- Секции через 120px, паддинг карточек 24px — просторный «соборный» ритм.
- Текстовая лестница `#d8ecf8` → `#d1e4fa` → `#c7d3ea` → `#9da7ba`.
- Conic-spotlight у верха каждого full-bleed hero.

DON'T:
- Не вводить дополнительные хроматические акценты — палитра монохромная с одним фиолетовым CTA (плюс светофор в отчётах и блоках результатов, см. §2.1).
- Не использовать сплошные цветные бордеры.
- Не делать display-заголовки жирнее 500.
- Не применять обычные drop-shadow.
- Не смешивать радиусные семейства.
- Не класть `#ffffff` на фоны ярче `rgba(186,214,247,0.12)`.
- Не использовать Skywash-градиент на body-тексте и кнопках — только крупнейшие display-заголовки (H1 хаба).
- Не использовать фотографии, стоковые иллюстрации, 3D — визуал строится из самих UI-элементов (формы, карточки, кольца баллов) как «стеклянных прототипов в тёмной студии» + line-art иконок.
- Никакой светлой темы — тема одна, тёмная.

### 3. Постраничные спецификации

Нужны макеты 6 сущностей. Все тексты ниже — смысловые заглушки: держись их смысла, длину и формулировки можешь слегка варьировать, язык — русский.

Скоуп: посадочные `/seo/full-audit/`, `/seo/pricing/`, `/seo/uptime-monitor/`, `/seo/ssl-monitor/`, `/seo/seo-monitor/` — ВНЕ скоупа этого пакета (Фазы 2–3): макеты для них не нужны, позже они соберутся из тех же компонентов. Ссылки на них в макетах («Тарифы» в header Б, кнопки «Подробнее» в тарифных карточках, paywall «Заказать полный аудит», пункты футера «Полный аудит» / «Мониторинг» / «Тарифы SEO») — обычные ссылки/кнопки, страницы-назначения рисовать не нужно.

#### 3.a. Обновлённая главная юр-сервиса `/` — минимальные изменения

Существующая главная НЕ перерисовывается. Её текущая структура (сохраняется как есть): hero «Узнайте, что нарушает ваш сайт — раньше, чем это найдёт робот РКН» с формой скана (URL-инпут + два обязательных чекбокса согласий + фиолетовый CTA «Проверить сайт»), секция «21 проверка по актуальным нормам» (6 юридических карточек), «Как это работает» (3 шага), «Тарифы» (0 ₽ / 700 ₽), футер с юрдоками.

Изменения РОВНО ДВА:

1. **Header:** в навигацию добавляется один пункт «SEO-аудит» (ссылка на `/seo/`). Итоговый nav страниц А: «Как работает / Тарифы / SEO-аудит / Проверить сайт».
2. **Один кросс-селл-блок перед футером** (после секции «Тарифы», до footer). Компактный: одна узкая центрированная GlassCard (feature) шириной ~800px. Содержимое на desktop: слева маленькая круглая иконка-тайл (line-art спидометр/график) и текст — заголовок heading-sm 24px «Проверьте и техническое здоровье сайта» + одна строка `#c7d3ea`: «SEO, скорость загрузки, SSL и безопасность — бесплатный экспресс-аудит за 30 секунд»; справа кнопка ghost (НЕ фиолетовая — фиолетовый на этой странице принадлежит юр-форме) «Запустить SEO-аудит →». На mobile кнопка переносится вниз под текст. Блок сдержанный, ниже по визуальному весу, чем секция тарифов: он не должен конкурировать с основной воронкой.

Покажи на макете только header + низ страницы (тарифы → кросс-блок → футер); середину можно обозначить схематично.

#### 3.b. Лендинг-хаб SEO-сервиса `/seo/`

Назначение: главная посадочная нового сервиса; и лендинг, и каталог инструментов одновременно. Целевой пользователь: владелец сайта / маркетолог / веб-мастер, пришедший по запросам «проверка сайта онлайн», «SEO-аудит сайта». H1: «Проверьте сайт онлайн: SEO, скорость, SSL и безопасность».

Порядок секций сверху вниз:

1. **Header силоса Б** (см. 3.f) — свой вариант навигации.

### 4. Правила иерархии страниц

- У каждой страницы **ровно один H1**. На хабе `/seo/` H1 — display 48px со Skywash; на страницах инструментов H1 — heading-lg 44px обычным `#d8ecf8` (без градиента); на отчёте H1 — домен проверенного сайта в шапке отчёта, heading 28px `#d8ecf8` (осознанное исключение, см. 3.e).
- **Skywash-градиент** разрешён только на display-заголовке (H1 хаба). Никогда — на H2/H3, body, кнопках, цифрах и wordmark в header (wordmark — Inter 16px `#d1e4fa`, см. 3.f).
- **Eyebrow** стоит НАД каждым H1 (исключение — отчёт: над доменом в шапке eyebrow не ставим) и над заголовком каждой секции; заголовки секций верхнего уровня — heading-lg 44px по центру; heading 28px — только для шапки отчёта и вложенных подсекций; под заголовком одна строка подзаголовка max 640px.
- Заголовки внутри карточек — subheading 18px или heading-sm 24px, Inter или Space Grotesk 500; вниз по вложенности типографика только уменьшается.
- **Один фиолетовый CTA на экран**: в hero — submit формы; на отчёте — кнопка paywall. Все прочие действия — ghost/outlined pill.
