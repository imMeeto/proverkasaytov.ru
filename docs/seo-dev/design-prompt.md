## НАЧАЛО ПРОМТА

### 1. Контекст и железное правило

Ты — продуктовый дизайнер. Проектируешь макеты для сайта «Проверка сайтов» (домен **проверкасайтов.рф**, язык русский, рынок РФ). На сайте живут **два сервиса**, оба — в одной визуальной системе:

- **Сервис А (существует, менять НЕ надо)** — юридическая проверка сайта на соответствие законам РФ: робот сканирует до 10 страниц, 21 проверка (152-ФЗ, cookie, оферта, авторизация-2026 и т.д.). Бесплатно — балл из 100, светофор, 2 критичных нарушения; полный отчёт 700 ₽. Его лендинг, экран отчёта, оплата и блог уже построены (см. приложенные макет и скриншоты).
- **Сервис Б (новый — его проектируешь ты)** — SEO/технический аудит сайта: бесплатный экспресс-аудит страницы (25+ проверок: мета-теги, Core Web Vitals, SSL, security-заголовки, robots.txt, sitemap и др.), каталог отдельных инструментов, платный полный аудит и мониторинг. Живёт в подкаталоге **`/seo/`** того же сайта.

**Настроение:** «тёмная стеклянная плата под током» — глубокий сине-фиолетовый холст `#161b36`, фростед-стеклянные карточки, тонкая blueprint-сетка на фоне, светящийся текст, единственный тёплый акцент — оранжевый как сигнал тревоги/риска, и фиолетовый как акцент действия. По фону бегут анимированные «дорожки печатной платы», соединяющие блоки. Просторный, «инженерно-соборный» ритм.

**ЖЕЛЕЗНОЕ ПРАВИЛО.** Дизайн-система уже существует и построена. Новые страницы обязаны **в точности** повторять её: те же фон, шрифты, цвета, радиусы, тени, компоненты и мотивы. НЕ вводи новых цветов, шрифтов, радиусов, типов теней, «улучшенных» градиентов. Всё нужное дано ниже и в приложенных материалах. Любое отклонение ломает систему и будет отклонено.

| Роль | Шрифт | Применение |

#### 2.3. Радиусы — четыре семейства, не смешивать

Кнопки/пилюли/переключатели — **999px**. Карточки/модалки — **16px**. Инпуты — **8px**, бейджи — **6px**. Круглые иконочные контейнеры — **9999px**.

#### 2.4. Тени и elevation — inset-фрост + внутреннее свечение + тёмный внешний ореол (без обычных drop-shadow)

- **Hairline (вместо бордера):** `inset 0 0 0 1px rgba(186,215,247,0.12–0.17)`.
- **Glass-карточка:** `rgba(216,236,248,0.14) 0 1px 1px inset, rgba(186,215,247,0.17) 0 0 0 1px inset, rgba(0,0,0,0.5) 0 20px 40px -14px`.
- **Premium-карточка (форма, тарифы, отчёт):** `rgba(216,236,248,0.22) 0 1px 1px inset, rgba(168,216,245,0.08) 0 24px 48px inset, rgba(186,215,247,0.16) 0 0 0 1px inset, rgba(0,0,0,0.55) 0 24px 48px`.
- **Фиолетовый bloom** (у premium-CTA-карточек и главной кнопки): добавить `rgba(102,58,243,0.15–0.28) 0 0 56–64px` и/или `0 0 28px rgba(102,58,243,0.45)` на самой кнопке.

#### 2.5. Фон страницы (общий слой атмосферы позади всех секций)

1. **Blueprint-сетка:** линии 1px `rgba(186,215,247,0.07)`, ячейка 80×80px, маска-затухание `radial-gradient(ellipse 90% 70% at 50% 0%, #000 30%, transparent 100%)`.

#### 2.6. Фирменные мотивы (обязательно повторить — это ДНК сайта)

- **Eyebrow-разделитель:** JetBrains Mono uppercase 15px, `letter-spacing:0.1em`, цвет `#c7d3ea`, по центру, с двух сторон — тонкие 1px-линии с градиентом от `transparent` к `rgba(186,215,247,0.12)`. Каждая секция открывается им.
- **Анимированные коннекторы-«дорожки»:** между секциями — маленький SVG (~240×110), где от центральной точки расходятся 3 «дорожки печатной платы»; по ним бегут светящиеся штрихи (фиолетовый `#663af3` + голубой `#b6d9fc`, drop-shadow-glow, анимация `traceFlow`). Это визуальный «шов» между блоками.
- **Схема-плата (hero-иллюстрация):** центральный «чип» с иконкой-щитом, от него дорожками к ряду «чипов»-плашек (у Сервиса А — законы, у Б — категории проверок), по дорожкам бегут штрихи.
- **Premium-карточки с «угловыми искрами»:** в 4 углах — точки 4px `#d8ecf8` opacity .5 с glow `0 0 6px rgba(216,236,248,0.9)`.
- **Форма ввода:** eyebrow над инпутом заканчивается **мигающим фиолетовым курсором** (блок 8×14px, анимация `blinkDot`); сам инпут-контейнер мягко пульсирует фиолетовым свечением (`inputPulse`), слева в инпуте — line-art иконка (лупа/глобус).
- **Пара «заголовок → подзаголовок-тревога»:** под крупным skywash-H1 (Dela Gothic, uppercase) — строка **оранжевым** `#e46d4c`, uppercase, Dela Gothic, с суммой/угрозой. Это фирменный приём Сервиса А; в Б применяй умеренно (риск ≠ штраф, а «проблемы/потери трафика»).
- **Hover:** кликабельные карточки/кнопки поднимаются `translateY(-2…-3px)` + лёгкое осветление фона; переходы `0.3s cubic-bezier(0.2,0.7,0.2,1)`.
- **Анимации:** `fadeUp` (появление экрана), `traceFlow`, `blinkDot`, `inputPulse`. Без светлой темы — тема одна, тёмная.

### 4. Переключатель сервисов (важно — спроектируй его)

На сайте два сервиса, и между ними должен быть **заметный переключатель-тумблер** — сегментированный pill-контрол в шапке, присутствует на **обоих** силосах:

- Два сегмента: «**По законам РФ**» и «**SEO-аудит**». Активный сегмент — фон `rgba(102,58,243,0.35)`, текст белый; неактивный — прозрачный, `#9da7ba`. Геометрия — pill 999px, как навигация в шапке.
- Клик = переход на реальный URL (`/` ↔ `/seo/`) с плавным переходом страницы (view-transition). Это **навигация между двумя страницами**, а НЕ подмена панели на одной.
- На главной Сервиса А переключатель живёт на уровне навигации в шапке — **не** как гигантский hero-элемент, не перегружая юридическую воронку.
- Покажи оба состояния тумблера (активны «По законам РФ» на `/` и «SEO-аудит» на `/seo/`).

Плюс — по одному сдержанному **кросс-селл-блоку** перед футером на каждой стороне (см. §5.7 и обновление главной ниже).

### 6. Правила иерархии

- Ровно один H1 на страницу. Хаб `/seo/` — display + skywash; страницы инструментов — heading-lg `#d8ecf8` без градиента; отчёт — домен heading 28px (исключение, eyebrow не ставим). Skywash — только на display-H1 хаба и в логотипе, никогда на H2/body/кнопках/числах.
- Eyebrow — над каждым H1 (кроме шапки отчёта) и над каждой секцией.
- Один фиолетовый CTA на экран. Все прочие действия — ghost/outlined pill. Оранжевый — только сигнал риска/красный вердикт, не кнопка.

### 7. Формат выдачи

Статичные макеты в двух ширинах: **desktop** (контент 1200px, фрейм 1440px с тёмными полями) и **mobile 390px**. Список: (1) обновлённая `/` — шапка с тумблером + низ страницы; (2) хаб `/seo/` целиком с каталогом (фильтр «Все», все 9 карточек); (3) шаблон инструмента с состоянием результата + врезки loading/ошибки; (4) SEO-отчёт + врезки прогресса/ошибки; (5) header Б с раскрытым дропдауном + mobile-меню (бургер → полноэкранная deep-glass панель); (6) тумблер-переключатель в обоих состояниях. На mobile: одна колонка, сетка карточек 1-в-ряд, фильтры — горизонтальный скролл, ScoreRing по центру, ритм секций сжимается. Всё — строго в приведённой системе и в стиле приложенных макета/скриншотов.
