# ПС-02 · Схема БД (PostgreSQL 17 + Drizzle)

Принцип: храним минимум персональных данных. Email — только если пользователь пошёл в оплату. IP — только хеш для rate-limit (живёт в Redis, в PG не пишется).

## 1. Таблицы

```ts
// enums
export const scanStatus = pgEnum('scan_status', ['queued', 'running', 'done', 'failed']);
export const checkStatus = pgEnum('check_status', ['pass', 'warn', 'fail', 'unable', 'not_applicable']);
export const severity = pgEnum('severity', ['critical', 'major', 'minor']);
export const paymentStatus = pgEnum('payment_status', ['pending', 'succeeded', 'canceled', 'refunded']);

export const scans = pgTable('scans', {
  id: uuid('id').primaryKey().defaultRandom(),        // = публичная ссылка на отчёт
  url: text('url').notNull(),                          // нормализованный исходный URL
  domain: text('domain').notNull(),                    // eTLD+1, для дедупа
  status: scanStatus('status').notNull().default('queued'),
  score: integer('score'),                             // 0..100, null пока не done
  isPaid: boolean('is_paid').notNull().default(false),
  email: text('email'),                                // появляется при оплате
  meta: jsonb('meta').$type<ScanMeta>(),               // см. §2
  error: text('error'),
  createdAt: timestamp('created_at').notNull().defaultNow(),
  startedAt: timestamp('started_at'),
  finishedAt: timestamp('finished_at'),
});

export const checkResults = pgTable('check_results', {
  id: serial('id').primaryKey(),
  scanId: uuid('scan_id').notNull().references(() => scans.id, { onDelete: 'cascade' }),
  checkId: text('check_id').notNull(),                 // 'https_ssl', 'forms_consent_checkbox'...
  status: checkStatus('status').notNull(),
  severity: severity('severity').notNull(),
  evidence: jsonb('evidence').$type<Evidence[]>(),     // см. §3
  createdAt: timestamp('created_at').notNull().defaultNow(),
}, (t) => [uniqueIndex('uq_scan_check').on(t.scanId, t.checkId)]);

export const payments = pgTable('payments', {
  id: uuid('id').primaryKey().defaultRandom(),
  scanId: uuid('scan_id').notNull().references(() => scans.id),
  provider: text('provider').notNull().default('yookassa'),
  externalId: text('external_id').notNull(),           // id платежа у провайдера
  amount: integer('amount').notNull(),                  // в копейках: 70000
  status: paymentStatus('status').notNull().default('pending'),
  email: text('email').notNull(),
  receiptUrl: text('receipt_url'),
  raw: jsonb('raw'),                                    // последний webhook как есть
  createdAt: timestamp('created_at').notNull().defaultNow(),
  updatedAt: timestamp('updated_at').notNull().defaultNow(),
}, (t) => [uniqueIndex('uq_payment_external').on(t.provider, t.externalId)]);

export const emailLog = pgTable('email_log', {
  id: serial('id').primaryKey(),
  scanId: uuid('scan_id').references(() => scans.id),
  to: text('to').notNull(),
  type: text('type').notNull(),                        // 'report' | 'payment_failed'
  status: text('status').notNull(),                    // 'sent' | 'error'
  error: text('error'),
  createdAt: timestamp('created_at').notNull().defaultNow(),
});
```

## 2. Тип `ScanMeta`

```ts
type ScanMeta = {
  ip?: string;              // IP проверяемого сайта (не пользователя!)
  geo?: { country: string; asn?: string; provider?: string };
  ssl?: { valid: boolean; validTo?: string; daysLeft?: number; issuer?: string };
  pagesCrawled?: { url: string; status: number; ms: number }[];
  loadMs?: number;          // главная страница
  innFound?: string | null; // ИНН из футера, для чека РКН
  consents: { pd: true; ownership: true };  // чекбоксы пользователя, фиксируем факт
  userAgentUsed: string;
};
```

## 3. Тип `Evidence`

Единый формат находки для всех чеков — из него строится и веб-отчёт, и PDF:

```ts
type Evidence = {
  pageUrl: string;          // где нашли
  detail: string;           // человекочитаемо: 'Форма #callback: поля name, phone — без чекбокса согласия'
  selector?: string;        // CSS-селектор элемента, если применимо
  extra?: Record<string, string | number>; // домен трекера, срок серта и т.п.
};
```

**[Правовой ресёрч 07.2026] Минимизация ПДн третьих лиц.** `evidence.detail`/`extra` описывают *технические признаки* (наличие поля `phone`, селектор формы, домен трекера), но НЕ фактические значения ПДн третьих лиц (реальные ФИО/телефоны/email с проверяемого сайта) — пайплайн маскирует их regex-ом до записи. Сырой HTML (`PageSnapshot.html`) живёт только в памяти скана, в БД не сохраняется. Обоснование — docs/legal-research-2026-07.md, ПС-08 §8.

## 4. Индексы и правила

- `scans(domain, created_at DESC)` — дедуп: последний `done`-скан домена младше 60 минут возвращается вместо нового.
- `payments(provider, external_id)` UNIQUE — идемпотентность webhook.
- `check_results(scan_id, check_id)` UNIQUE — повторный прогон чека перезаписывает через upsert.
- Каскадное удаление `check_results` при удалении скана; `payments` — RESTRICT (финансовые записи не удаляем).

## 5. Ретенция (соответствие собственной политике ПДн)

Cron-job в воркере, раз в сутки:
- неоплаченные сканы старше 30 дней → удаляются вместе с результатами; **[правовой ресёрч 07.2026: срок снижен 90→30, минимизация экспозиции по 152-ФЗ ст. 5 ч. 7; docs/legal-research-2026-07.md]**
- оплаченные сканы старше 365 дней → `email` затирается (`null`), отчёт остаётся доступен по ссылке;
- `email_log` старше 365 дней → удаляется.

Сроки должны совпадать с текстом политики ПДн сервиса (ПС-09 §2) — если владелец меняет одно, меняет и второе.
