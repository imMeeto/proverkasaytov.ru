# ПС-05 · API, фронт, SSE

## 1. API-роуты

Все входы — Zod `.strict()`. Все ответы — `{ ok: true, data } | { ok: false, error: { code, message } }`.

### POST /api/scan
```ts
body: { url: string, consentPd: literal(true), consentOwnership: literal(true), captchaToken: string }
```
Пайплайн хендлера: SmartCaptcha-верификация (`security/captcha.ts`) → нормализация URL (добавить https:// если нет схемы; выделить eTLD+1) → SSRF-предпроверка (ПС-08 §2) → rate-limit `scan:create` по IP: 5/сутки (ПС-08 §5) → дедуп: последний `done`-скан того же домена младше 60 мин → вернуть его id с флагом `deduped: true` → иначе INSERT + enqueue → `{ scanId }`.

### GET /api/scans/[id]
Возвращает scan + результаты. **Усечение неоплаченного** делается на сервере, не CSS-блюром:
```ts
if (!scan.isPaid) {
  // fail-результаты сортируются по penalty; первые 2 отдаются целиком
  // остальные fail/warn: { checkId, title, status, severity } — без evidence и текстов
  // pass/unable/not_applicable отдаются целиком (это бесплатная ценность и честность)
}
```
Тексты «что делать» и сниппеты подмешиваются из `checks/texts.ts` только для разблокированных элементов. Клиент физически не получает платный контент до оплаты.

### GET /api/events/scan/[id]
SSE. Заголовки `text/event-stream`, `no-cache`. Подписка на EventEmitter-канал `scan:{id}` (fan-out от singleton Redis-subscriber). Heartbeat `: ping` каждые 25 с. При `stage: done|failed` — отправить финальное событие и закрыть.

### POST /api/payments
```ts
body: { scanId: uuid, email: string.email() }
```
Проверки: скан существует, `status='done'`, `isPaid=false`, нет pending-платежа младше 15 мин (иначе вернуть его же confirmationUrl). → `PaymentProvider.create()` (ПС-06) → INSERT payments(pending) → `{ confirmationUrl }`.

### POST /api/webhooks/payment
См. ПС-06 §4. Без captcha и rate-limit по IP пользователя (это ЮKassa), но с проверкой подлинности.

### GET /report/[id]/print?token=...
Серверная страница для PDF. Доступ: `token === HMAC_SHA256(scanId, REPORT_PRINT_SECRET)`. Рендерит полный отчёт в печатной вёрстке (без блюра — вызывается только воркером и только для оплаченных).

## 3. Страница отчёта `/report/[id]` — состояния

1. **queued/running**: прогресс-лента по SSE-событиям («обхожу страницу 3 из 8…»), скелетон отчёта. Фолбэк без SSE — поллинг `GET /api/scans/[id]` раз в 3 с.
2. **failed**: честное сообщение + причина (если безопасна: «сайт не отвечает») + кнопка «Проверить заново».
3. **done, не оплачен**: 
   - шапка: балл-кольцо, светофор-вердикт, счётчики 🔴/🟡/🟢, домен, «проверено N страниц»;
   - блок «Чем грозит»: суммы штрафов по найденным fail (из `texts.ts`) с оговоркой про информационный характер;
   - 2 открытых нарушения — карточки целиком: что нашли (evidence) / что делать / статья;
   - остальные — заголовок + статус + замок, тело: серые плейсхолдеры (не настоящий текст под CSS-блюром!);
   - выполненные (🟢) — открытый список;
   - CTA-блок: email-поле + кнопка «Открыть полный отчёт — 700 ₽» + буллеты (все нарушения, сниппеты, PDF на почту) + ссылка на оферту.
4. **done, оплачен**: всё открыто, у карточек — вкладка «Готовый код» с copy-кнопкой, сверху кнопка «Скачать PDF» (presigned-ссылка), плашка «отчёт отправлен на a***@…».

## 4. UI-правила

- shadcn/ui + Tailwind. Шрифты **только self-hosted** (`@font-face`, woff2 в `/public/fonts`) — рекомендация: Inter или Onest с variable-начертанием.
- Яндекс.Метрика подключается **только после клика «Принять»** в собственном cookie-баннере (реализация — свой же сниппет из `texts.ts`; баннер с равнозначной кнопкой «Отклонить»). Цели: `scan_started`, `scan_done`, `pay_click`, `pay_success`.
- Никаких внешних CDN: все зависимости — из бандла.
- Мобильная вёрстка приоритетна: форма скана и отчёт должны быть полноценны на 360 px.
- Балл-кольцо, светофор и карточки — простые компоненты, без сторонних чарт-библиотек.

## 5. SEO-минимум

SSR всех публичных страниц, `metadata` API: title/description на лендинг и юрстраницы. Страница отчёта — `robots: noindex` (приватная ссылка). OG-картинка лендинга — статичная. Блог/SEO-статьи — вне MVP.
