# ПС-07 · PDF и email

## 1. Генерация PDF — тем же Playwright

Воркер уже несёт Chromium — отдельная PDF-библиотека не нужна.

Job `deliver {scanId}`:
1. Проверить `scans.isPaid === true` (иначе выйти с логом — защита от случайного enqueue).
2. `token = HMAC_SHA256(scanId, REPORT_PRINT_SECRET)`.
3. `page.goto('{APP_INTERNAL_URL}/report/{id}/print?token=' + token)` — воркер ходит в app по внутренней Docker-сети (`http://app:3000`), не через публичный домен.
4. `await page.pdf({ format: 'A4', printBackground: true, margin: {top:'14mm', bottom:'14mm', left:'12mm', right:'12mm'} })`.
5. Upload в S3: ключ `reports/{scanId}.pdf`, bucket приватный.
6. Presigned URL на 30 суток → письмо. Повторная генерация ссылки — кнопкой на странице оплаченного отчёта (`GET /api/scans/[id]/pdf-url`, отдаёт свежий presigned).

## 2. Печатная вёрстка `/report/[id]/print`

- Без интерактива, блюра, SSE и Метрики. Полный отчёт: шапка с баллом → «чем грозит» → все нарушения с evidence, «что делать» и сниппетами (сниппеты — моноширинным блоком, переносы строк сохранить) → выполненные пункты → страница-дисклеймер.
- Колонтитул: `{SITE_NAME} · отчёт по {domain} · {дата} · стр. X` через `@page` CSS.
- Обязательный текст на титуле и в подвале: «Отчёт носит информационный характер и не является юридической консультацией».
- `page-break-inside: avoid` на карточках нарушений.

## 3. Email

Модуль `services/mailer.ts`: nodemailer поверх SMTP из env. Провайдер — российский транзакционный SMTP (рекомендация владельцу: RuSender / Unisender Go / NotiSend; выбор не влияет на код). Обязательно: SPF, DKIM, DMARC на домене отправителя — иначе письма о «проверке законности» лягут в спам, что иронично.

### Письмо `report` (после оплаты)
- Тема: `Полный отчёт по {domain} — {score}/100`
- Тело (простая HTML-вёрстка, инлайн-стили, без внешних картинок): вердикт-строка, кнопка «Открыть отчёт» → `/report/{id}`, ссылка на PDF (presigned), строка «чек об оплате придёт отдельным письмом от платёжного сервиса».
- Вложение: PDF, если размер < 7 МБ; иначе только ссылка.
- Запись в `email_log`; ошибка отправки → ретрай job (3 попытки), после — `email_log.status='error'`, отчёт всё равно открыт по ссылке (доставка письма не блокирует доступ).

### Письмо `payment_failed` — не отправляем (статус виден на странице). Никаких маркетинговых рассылок из транзакционного контура: согласия на рекламу мы не собираем — и сами же за это штрафуем.
